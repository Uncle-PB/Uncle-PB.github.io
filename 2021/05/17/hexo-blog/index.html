更多 Hexo 命令

常见命令
hexo new “postName” #新建文章
hexo new page “pageName” #新建页面
hexo generate #生成静态页面至 public 目录
hexo server #开启预览访问端口（默认端口 4000，’ctrl + c’ 关闭 server）
hexo deploy #部署到 GitHub
hexo help #查看帮助
hexo version #查看 Hexo 的版本

缩写
hexo n == hexo new
hexo g == hexo generate
hexo s == hexo server
hexo d == hexo deploy

组合命令
hexo s -g #生成并本地预览
hexo d -g #生成并上传

GitHub Pages 静态部署

参考 GitHub Pages 功能
1、首先在 GitHub 创建用户名同名的 Repository，便于直接使用 username.github.io (例如：Uncle-PB.github.io ) 进行访问

2、Setting -》Options -》Danger Zone 选择 Change visibility

3、Setting -》Pages 可以看到结果，也可设置其他选项

4、配置 SSH
本地代码提交到 GitHub 需要拥有提交权限，直接配置用户密码不太安全，采用配置 SSH 的方式来解决本地和服务器连接问题
检查本机已存在的 ssh 密钥，如果提示：No such file or directory 说明你是第一次使用 git。

5、生成 RSA 密钥

6、然后连续 3 次回车，最终会生成一个文件在用户目录下，打开用户目录，找到.ssh\id_rsa.pub 文件，记事本打开并复制里面的内容，打开你的 github 主页，进入个人设置 -> SSH and GPG keys -> New SSH key： ，将刚复制的内容粘贴到 key 那里，title 随便填，保存

7、测试是否成功 (地址不用改，直接复制)

如果提示 Are you sure you want to continue connecting (yes/no)? 输入 yes，然后会看到

“Hi Uncle-PB! You’ve successfully authenticated, but GitHub does not provide shell access.”

看到这个信息说明 SSH 已配置成功！
