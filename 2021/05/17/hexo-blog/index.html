    Hexo + GitHub 搭建个人博客

    虽然每次需通过MakDown文件生成HTML文件，但这种方式更加自由干净且没有广告。

    附上一些参考资料相关地址：Hexo 官方文档 Fluid 官方文档 Fluid (GitHub) Fluid 官方博客

    环境准备

    更多详细使用介绍，最好参考官方文档了解具体的配置、命令、写作 等。
    Hexo 是一个基于nodejs 的静态博客网站生成器,使用之前的环境准备需要安装以下程序。

    • Node.js (Node.js 版本需不低于 10.13，建议使用 Node.js 12.0 及以上版本)
    • Git

    安装Hexo

    Node 和 Git 安装完毕后，就可以通过npm命令下载安装Hexo。
    首先我们建立一个目录文件夹 MyBlog 右键启动Git Bash Here，然后输入命令(需要权限的话 sudo 执行)：

    $ npm install -g hexo-cli

    完成后初始化在本地生成Hexo相关目录：

    $ hexo init

    启动Hexo服务 或者 hexo s (-p 端口：默认4000)：

    $ hexo server

    本地访问：

    localhost:4000

    至此，Hexo已经安装完成并可以使用了，只是样式上并不怎么美观，我们可以去官网下载相应的主题。

    更多Hexo 命令

    常见命令
    hexo new “postName”    #新建文章
    hexo new page “pageName”    #新建页面
    hexo generate    #生成静态页面至public目录
    hexo server    #开启预览访问端口（默认端口4000，’ctrl + c’关闭server）
    hexo deploy    #部署到GitHub
    hexo help    #查看帮助
    hexo version    #查看Hexo的版本

    缩写
    hexo n == hexo new
    hexo g == hexo generate
    hexo s == hexo server
    hexo d == hexo deploy

    组合命令
    hexo s -g    #生成并本地预览
    hexo d -g    #生成并上传

    主题下载和安装

    Hexo官网上提供了丰富的主题可选，你只需要打开对应的界面（https://hexo.io/themes/ ）选择喜欢的，然后点击名称跳转到GitHub仓库选择下载或者克隆对应的zip文件到本地，并且解压到网站目录下的themes目录即可。

    1、这里介绍下本博客使用的主题 Fluid 的安装过程，直接使用命令npm下载：

    $ npm install --save hexo-theme-fluid

    2、然后在博客目录下创建 _config.fluid.yml，将主题的 _config.yml （从GitHub复制即可）内容复制进去

    3、指定主题，修改 Hexo 博客目录中的 _config.yml：

    theme: fluid  # 指定主题
language: zh-CN  # 指定语言，会影响主题显示的语言，按需修改

    4、这样你的主题就应用好了，然后重新执行 hexo g 来重新生成。如果出现一些莫名其妙的问题，可以先执行 hexo clean 来清理一下public的内容，然后再来重新生成和发布。

    GitHub Pages 静态部署

    参考GitHub Pages功能
    1、首先在GitHub创建用户名同名的Repository，便于直接使用 username.github.io (例如：Uncle-PB.github.io )进行访问

    2、Setting -》Options -》Danger Zone 选择Change visibility

    3、Setting -》Pages 可以看到结果，也可设置其他选项

    4、配置SSH
    本地代码提交到GitHub 需要拥有提交权限，直接配置用户密码不太安全，采用配置SSH的方式来解决本地和服务器连接问题
    检查本机已存在的ssh密钥，如果提示：No such file or directory 说明你是第一次使用git。

    $ cd ~/.ssh

    5、生成RSA 密钥

    $ ssh-keygen -t rsa -C "240***@**.com"

    6、然后连续3次回车，最终会生成一个文件在用户目录下，打开用户目录，找到.ssh\id_rsa.pub文件，记事本打开并复制里面的内容，打开你的github主页，进入个人设置 -> SSH and GPG keys -> New SSH key： ，将刚复制的内容粘贴到key那里，title随便填，保存

    7、测试是否成功(地址不用改，直接复制)

    $ ssh -T git@github.com

    如果提示Are you sure you want to continue connecting (yes/no)? 输入yes，然后会看到

    “Hi Uncle-PB! You’ve successfully authenticated, but GitHub does not provide shell access. “

    看到这个信息说明SSH已配置成功！

    8、修改Hexo 的 _config.yml配置 ‘deploy’ 参数

    type: git
  repository: git@github.com:Uncle-PB/Uncle-PB.github.io.git
  branch: master

    9、安装hexo git插件部署 

    $ npm install hexo-deployer-git --save

    10、发布部署

    $ hexo d

    11、访问博客地址

    Https://Uncle-PB.github.io
