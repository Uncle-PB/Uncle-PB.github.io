    使用多个@RequestBody

    发布时间 : 2021-07-13 14:42
    阅读 :
    1. 只能使用一个 @RequestBody
    2. 改进支持多个 @RequestBody
      1. ServletInputStreamCopier
      2. RequestWrapper
      3. 在Filter中放入wrapper替代servletRequest
    3. 参考

    如何在Controller中使用多个@RequestBody注解去获取body中的数据呢？

    只能使用一个 @RequestBody

    由于对HttpServletRequest 进行读取时，getInputStream()方法只能调用一次(认为body中的参数只需一次获取全部就能满足调用需求)，这样导致我们在接口中使用多个 @RequestBody 注解的话是会报错的。IOException: Stream closed

    如果不想要一个大对象获取所有的入参的话，可以使用 Map 的方式装载，想要什么参数就从map中get，但这种方式也不太优雅。下面的方法介绍通过代理模式修改getInputStream()方法，每次获取返回一个新的流，从而支持多个 @RequestBody 的使用。

    改进支持多个 @RequestBody

    ServletInputStreamCopier

    自定义类继承 ServletInputStream，通过构造方法传入需要复制的byte[] 数组，这样每次返回自定义的对象都能获取到body中的内容。

    代码 
    import org.apache.catalina.connector.InputBuffer;

import javax.servlet.ReadListener;
import javax.servlet.ServletInputStream;
import java.io.IOException;
import java.nio.ByteBuffer;

public class ServletInputStreamCopier extends ServletInputStream {
    private final InputBuffer buf = new InputBuffer();

    public ServletInputStreamCopier(byte[] bodyCopier) {
        buf.setByteBuffer(ByteBuffer.wrap(bodyCopier));
    }

    @Override
    public boolean isFinished() {
        return buf.isFinished();
    }

    @Override
    public boolean isReady() {
        return buf.isReady();
    }

    @Override
    public void setReadListener(ReadListener readListener) {
        buf.setReadListener(readListener);
    }

    @Override
    public int read() throws IOException {
        return buf.read();
    }
}

    RequestWrapper

    注意到我们编写ReReadableHttpServletRequest时，是从HttpServletRequestWrapper继承，
    而不是直接实现HttpServletRequest接口,这是因为，Servlet的每个新版本都会对接口增加一些新方法。
    从HttpServletRequestWrapper继承可以确保新方法被正确地覆写了，因为HttpServletRequestWrapper是由Servlet的jar包提供的，目的就是为了让我们方便地实现对HttpServletRequest接口的代理

    代码 
    import org.apache.commons.io.IOUtils;

import javax.servlet.ServletInputStream;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletRequestWrapper;
import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.io.UnsupportedEncodingException;

/**
 * 1、记录接口信息
 * 2、返回InputStream 支持多个@RequestBody
 */
public class RequestWrapper extends HttpServletRequestWrapper {
    private byte[] bodyCopier;

    public RequestWrapper(HttpServletRequest request) throws IOException {
        super(request);
        bodyCopier = IOUtils.toByteArray(request.getInputStream());
    }

    /**
     * 每次返回new ServletInputStream() 保证可重复获取流
     */
    @Override
    public ServletInputStream getInputStream() throws IOException {
        return this.bodyCopier == null ? null : new ServletInputStreamCopier(bodyCopier);

    }

    @Override
    public BufferedReader getReader() throws IOException {
        return new BufferedReader(new InputStreamReader(this.getInputStream()));
    }

    public String getBody() throws UnsupportedEncodingException {
        return this.bodyCopier != null ? new String(this.bodyCopier, "utf-8").replaceAll("\\n", "").replaceAll(" ", "") : null;
    }

}

    在Filter中放入wrapper替代servletRequest

    最后在过滤器中放入我们修改后的RequestWrapper就行了

    代码 
    import cn.dragon.framework.request.RequestWrapper;
import lombok.extern.slf4j.Slf4j;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

import javax.servlet.*;
import javax.servlet.http.HttpServletRequest;
import java.io.IOException;

/**
 * @author lhl
 * @date 2021/4/29
 * @desc 统计过滤器
 */
@Slf4j
public class RequestLogFilter implements Filter {
    private static final Logger logger = LoggerFactory.getLogger(RequestLogFilter.class);

    @Override
    public void init(FilterConfig filterConfig) throws ServletException {

    }

    @Override
    public void doFilter(ServletRequest servletRequest, ServletResponse servletResponse, FilterChain filterChain) throws IOException, ServletException {
        Long start = System.currentTimeMillis();
        HttpServletRequest request = (HttpServletRequest) servletRequest;
        RequestWrapper wrapper = new RequestWrapper(request);
        String uri = request.getRequestURI();
        logger.info("接口：{},入参：{}", uri, wrapper.getBody());
        filterChain.doFilter(wrapper, servletResponse);
        logger.info("接口：{},耗时：{}", uri, System.currentTimeMillis() - start);
    }

    @Override
    public void destroy() {

    }
}

    然后这样就可以愉快的使用多个 @RequestBody 注解了
